# AutoMoyu 🎣 — Minecraft 基岩版 半/全自动钓鱼助手

给基岩版（UWP）用的自动甩竿工具。UWP/DirectX 全屏对 `PostMessage` 之类的后台消息注入天生免疫，常见的模拟点击方案在这上面大多会失灵，核心思路因此是：

- **不用 PostMessage**，而是用 `SendInput` 从驱动层注入右键 —— 和 AutoHotkey 一样能打进 UWP/DirectX 全屏。
- **不保存任何参考图**。每次甩竿后当场重拍一张"基准帧"，之后只看画面**有没有变化**。所以鱼竿抚摸变色、背景变化、换分辨率都不会让它失灵，也不用每次重新截图。

## 依赖

Python 3.10+，需要 `mss` 和 `numpy`（常见环境一般已自带）。GUI 用自带的 tkinter，输入/热键用 ctypes 调 Win32，**不需要 pywin32**。

```
pip install -r requirements.txt
```

## 运行

- 双击 `run.bat`，或
- 命令行 `python run.py`

## 用法（半自动，最常用）

适用于已经搭建好半自动钓鱼机的情况（甩竿要手动、上钩自动收）。工具只负责在"鱼收上来、鱼竿空了"时自动再甩一次。

1. 进游戏，手持鱼竿，站在钓鱼机前。
2. 让经验条里有一点经验（非空），点 **「自动定位经验条」** → 有 3 秒切回 Minecraft → 工具自动框住那条绿色横条。
   - 也可以点 **「选择区域」** 手动拖框（经验条 = 框那条绿条；鱼钩 = 框住手持鱼竿那一小块）。
3. 模式选 **半自动**，目标选 **经验条**。
4. 按 **F6**（或点「开始」）→ 工具甩竿 → 监视经验条 → 一有变化(=钓到)就再甩。
5. 再按 **F6** 停止，**F8** 急停。

> **调灵敏度**：运行时看「实时检测」那条 меter。蓝条是当前变化量，越靠右越接近阈值；触发瞬间会变红并显示 ★。经验条方案几乎不会误触；如果偶尔漏检就把灵敏度调高一点。

## 全自动（实验性）

没有钓鱼机、直接找片水面时用。工具会甩竿 → 检测咬钩 → 收竿 → 再甩。咬钩检测靠浮漂区域的瞬间变化，受水面动画/遮挡影响，需要多调灵敏度和时序。目标建议选 **鱼钩/浮漂** 并框住浮漂附近。

## 经验条 vs 鱼钩，选哪个？

- **经验条**：推荐。UI 是静态的，钓到鱼一定涨经验，几乎零误触，通用性最好。唯一要求是别被聊天框/物品栏完全挡住。
- **鱼钩/浮漂**：通用差分检测，适合不想动经验条的场景，但鱼竿本身的轻微晃动可能带来误触，需要把灵敏度调低些。

两个都能在"目标"里切换，测试阶段可以各试一下看哪个准。

## 数据

- 本次 / 生涯的钓鱼条数、时长、时速，以及历史记录，都存在 `data/` 目录（`config.json` / `stats.json`）。
- 「历史记录」按钮看每场明细，「清空生涯」重置。

## 小贴士 / 排错

- **安全锁**：默认「仅当窗口标题含 Minecraft 时才点击」，切到其他窗口时不会误触发点击。窗口名可在选项里改（比如国际版是 `Minecraft`，某些版本可能不同，可用「预览」/日志确认）。
- **点了没反应**：确认 Minecraft 是前台聚焦窗口；确认它是**无边框窗口/窗口化全屏**（独占全屏可能截不到图，画面会全黑）。
- **想看报错**：把 `run.bat` 里的 `pythonw` 改成 `python`，或直接 `python run.py`。
- 首次自动定位失败，多半是经验条是空的（0 经验没有绿色）或被挡住，钓一条有经验后再定位，或改用手动框选。
